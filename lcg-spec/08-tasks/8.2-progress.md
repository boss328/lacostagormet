## Decisions made mid-build

- **Design system locked at v5 "cinematic" level (2026-04-18).** Authoritative spec is `01-context/1.3-design-system.md`. Key choices: Fraunces serif (display) + JetBrains Mono (labels) via next/font/google — NO Inter, Georgia, or system fonts. Warm mercantile palette with new hex values (ink #1A110A, paper #F6EEDE, brand-deep #4E2410, gold #B88A48, accent #C14828). Italic accent words in every headline (e.g., "Shop by *category*"). Catalog-style section numbering (§ I, § II, § III) with roman numerals. Staggered page-load animations (0.1s, 0.25s, 0.4s, 0.55s delays) and IntersectionObserver-based scroll reveals. Real photography with uniform warm color grading. Specific hover transitions defined per component (product cards: lift + dual shadow + image zoom; brand rows: accent bar draw + slide right; buttons: overlay-slide fill). Every page uses a named template (CatalogPage, ProductDetailPage, CartPage, CheckoutPage, AccountPage, AdminPage). Admin uses same design language, just denser. Phase 3 prompt rewritten in the Claude Code playbook to enforce all of this.
- **Final fulfillment model (post-Jeff-call, 2026-04-17):** one unified catalog, one `supplier` dropdown per product. Vendors table seeds: Houston's, Sunny Sky, Mocafe, David Rio, Monin, Torani, Kerry, DaVinci Gourmet, iBev, Smartfruit, Tiki Breeze, and a "Home/Garage" row with `is_self_fulfilled=true`. No per-warehouse routing. No label printing integration. Orders group by supplier → admin generates pre-filled email Jeff can edit → sends via Gmail API. Home/Garage items land in `/admin/ship-yourself` queue with no supplier email; Jeff prints labels via his existing USPS/UPS/PayPal workflow.
- **Jeff writes his own email templates.** Seeded with a basic placeholder template; will be replaced when Jeff sends real copy. Template variables: `{{order_number}}`, `{{customer_name}}`, `{{shipping_address}}`, `{{items_list}}`, `{{po_number}}`, `{{my_account_number}}`.
- **Tracking API deferred to Phase 2.** Jeff said "push back and forth if easy, Phase 2 if complex." We need to see real supplier reply emails before we can scope it. V1 = manual paste-back. Revisit 30-60 days post-launch once we have supplier reply data.
- **Zero sales tax in v1 — TaxJar is v2-optional.** Jeff's entire 158-SKU catalog is non-taxable grocery food. CA (his only nexus state) doesn't tax groceries. Build uses a pluggable `calculateTax()` function that returns `0` by default; swapping in TaxJar later is a 2-hour task if needed. Agreement has a clause making client responsible for notifying Builder if taxable products are added or nexus is crossed in a new state.
- **Amazon Storefront IS real and IS out of v1 scope.** Jeff runs an Amazon Seller Central storefront in addition to the website. Those orders currently ship via Houston's email handoff same as the website. Jeff wants Amazon orders eventually visible in the new admin, but Amazon SP-API is a multi-week integration (plus 1-3 weeks of Amazon's approval process) that would blow the weekend timeline and the $1,500 price. **V1 is website-only; Amazon stays in Seller Central as it does today.** Schema includes `orders.source` column and `order_source` enum so Phase 2 integration slots in cleanly. Phase 2 priced separately at $2,500-4,000 when the time comes.
- **Big Train concentration risk flagged.** Big Train + likely-Big-Train-miscategorized products account for the vast majority of revenue. The new admin should surface vendor diversification (what % of revenue comes from which vendor) as a metric Jeff can watch.
- **Catalog migration strategy:** prefer vendor spreadsheets as source of truth; use BC export only for product slugs (URL preservation), Jeff's custom descriptions, and product images.

## Issues encountered

- **BC image paths in the CSV aren't current CDN URLs (2026-04-18).** The Phase 2 migration stored `product_images.url` values like `k/997/Mango_Front__96453.jpg` — these are BigCommerce internal storage paths, not reachable URLs. The live site uses `/images/stencil/<size>/products/<pid>/<iid>/...` which cannot be derived from the CSV data we have. Need Jeff to re-export with a "Product Image URL" column, OR scrape URLs from the live `lacostagourmet.com`. Retainer task post-launch. **Workaround shipped:** `ImageWithFallback` swaps to a branded fallback tile (brand name + SKU on paper-2→paper-3 gradient for product cards, category name + item count for category tiles) whenever a next/image load 404s. Cards stay visually intact.
