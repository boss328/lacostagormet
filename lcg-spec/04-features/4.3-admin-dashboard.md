## /admin (Dashboard)

The "morning view" — grandpa opens this, sees what matters.

Components:
- **Today's revenue** (big number, with delta vs yesterday)
- **New orders** (count, with "View all →" link)
- **Unfulfilled orders** (count, with list of top 5)
- **Held for review** (Auth.net AFDS flagged, if any)
- **Last 7 days revenue** (line chart)
- **Top 5 products this week** (table)
- **Out-of-stock alerts** (if any product has `stock_status = 'out_of_stock'`)

No clever cards, no AI-generated summaries. Just numbers grandpa cares about.

## /admin/orders

### List view
- Filters: status (paid / held / cancelled / refunded), fulfillment status, date range, search by order number or customer email
- Columns: Order #, Date, Customer, Total, Status badge, Fulfillment badge, Vendors (icons), Actions
- Bulk actions v1: Export CSV. Later v2: bulk print packing slips.
- Pagination: 25 per page

### Detail view
Already speced in `4.2-vendor-fulfillment.md` — that screen is the heart of the admin. Refer there.

## /admin/products

### List view
- Filters: category, brand, vendor, status (active/inactive), stock status, has-image, has-description
- Columns: Image, SKU, Name, Brand, Vendor, Retail, Wholesale, Margin %, Status, Actions
- Bulk actions:
  - Activate / deactivate
  - Apply % price change
  - Change vendor assignment
  - Export selected to CSV
- "Import from spreadsheet" button → `/admin/products/import`

### Detail / edit view
- Main tab: Name, SKU, brand, category, description, short description, ingredients, pack size, weight, dimensions
- Pricing tab: retail price, wholesale cost, margin % (auto-calc), preferred vendor
- Images tab: drag-drop reorder, mark primary, alt text per image
- SEO tab: slug, meta title, meta description, canonical URL
- Audit tab: history of price changes, vendor reassignments, etc. (from `audit_log`)

### /admin/products/import
Spreadsheet importer — critical QoL feature.

- Drag-drop CSV or XLSX
- Select vendor context (which vendor sent this sheet)
- System auto-detects columns (SKU, name, price, etc.) based on saved mappings per vendor
- Preview table shows what'll be created vs updated vs skipped
- "Match by" dropdown: SKU (default), name, UPC
- "On conflict" options: update all fields, update prices only, skip existing
- "Dry run" button → shows what would happen without committing
- "Import" button → commits, shows progress, saves audit log entry

Save mapping templates per vendor so grandpa uploads Houston's weekly sheet and it just works:
- "Houston's Spring 2026 Pricing.xlsx" → system remembers "column B = SKU, column D = wholesale cost"

## /admin/vendors

- List of all vendors
- Per-vendor edit page with: contact info, email template, account number (for use in `{{my_account_number}}` variable), notes, `is_self_fulfilled` flag
- "Upload catalog" link → goes to products import with that vendor pre-selected

## /admin/customers

- List with filters (business / personal, signup date, total spend, last order)
- Detail view per customer: info, addresses, orders, saved payment methods (masked), lifetime value, notes
- Admin can:
  - Force password reset
  - Add admin notes
  - Apply a credit (future — v1 skips store credit)
  - Mark as VIP (shows a badge on future orders)

## /admin/finance/sales

- Date range picker (default: last 30 days)
- Charts:
  - Revenue over time (line)
  - Orders over time (line)
  - AOV over time (line)
- Tables:
  - Revenue by day (exportable CSV)
  - Revenue by brand
  - Revenue by vendor (grandpa's sales by which vendor supplied them)
  - Revenue by category

## /admin/finance/net-revenue

- Shows gross revenue, minus estimated Auth.net gateway fees (2.9% + $0.10), minus estimated WF interchange (~1.5% average, configurable)
- Date range selector
- "Reconciliation status" section:
  - For each Auth.net batch in the range, show the settlement in Auth.net vs the WF deposit (manual entry for now — v2 could auto-pull via bank API)
  - Flag discrepancies

**v1 version:** calculate estimated fees using a flat %. Label clearly as "estimated."
**v2:** pull actual fees from Auth.net Transaction Details API, reconcile against WF manual entries.

## /admin/finance/margin

- Per-product profit: retail - wholesale × units sold
- Table: SKU, Name, Units sold (period), Revenue, Wholesale cost, Gross margin $, Margin %
- Sortable by any column
- Filter by brand, vendor, category
- Bottom summary: total gross margin for period
- CSV export

**Honest caveat for grandpa:** margin only includes product cost vs sale price. It doesn't account for shipping costs eaten on free-ship orders, processing fees, returns. Label this as "Product margin" not "Net profit."

## /admin/finance/disputes

- Chargebacks + refunds list from Auth.net webhooks
- Columns: Date, Order, Amount, Reason, Response due, Status
- Click to see: original order, timeline, upload evidence (PDF or images) → stored in Supabase Storage
- "Mark as responded" button (grandpa handles actual response in Auth.net dashboard, this just tracks it here)

## /admin/finance/quickbooks

- "Last export" timestamp
- "Export now" button → generates Intuit-compatible file or pushes via QBO API
- Configurable: which accounts to map to (Sales, Shipping Income, Sales Tax Payable, etc.)
- Automated daily export at 2am (cron job) once config is set

**Important:** QBO mapping requires grandpa's input on his existing Chart of Accounts. V1 ships a "Configure mapping" wizard but the actual mapping is a TBD until grandpa gives us his QBO structure.

## /admin/blog

- Minimal CMS: list of posts, create/edit/delete
- Editor: Markdown or basic rich text
- Fields: title, slug, excerpt, cover image, body, meta, publish date, status (draft/published)
- No migration of existing BigCommerce blog posts v1 (there are 5-10, they're not valuable)

## /admin/settings

Tabs:
- **General:** site name, contact info, social links
- **Shipping:** free threshold, flat rate, HI/AK surcharge
- **Payment:** Auth.net credentials (view-only; edit via env vars for security)
- **Taxes:** TaxJar connection status, nexus states
- **Email:** Google Workspace OAuth status, Resend API key status (read-only UI, edit via env)
- **Vendors:** (link to vendor management)
- **Featured products:** drag-drop to set order on homepage
- **Banners:** optional homepage banner text ("Free shipping over $70!")
- **Fulfillment:** auto-send toggle, max order amount for auto-send
