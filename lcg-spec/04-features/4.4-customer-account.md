# 04.4 — Customer Account Dashboard

## Scope

Logged-in customer experience. Grandpa's existing site has a basic account page; we're building something better because he specifically identified reorder friction and tunnel vision as real problems.

## Routes

```
/account               → dashboard overview
/account/orders        → order history
/account/orders/[id]   → order detail
/account/addresses     → saved addresses
/account/payment-methods → saved cards (from Auth.net CIM)
/account/profile       → name, email, password, business info
```

### 3. Recommendations (the tunnel-vision breaker)

Three recommendation strips:

**"You might also like"** — products in the same brand/category as their most recent order. Example: "You buy a lot of Big Train Vanilla Chai — try our other Big Train flavors."

**"Back in stock"** — if any product they've ordered before recently went from `out_of_stock` to `in_stock`, surface it here.

**"New arrivals in your favorite brands"** — products added in last 90 days, filtered by brands they've ordered before.

**Recommendation algorithm (keep it simple):**

```ts
// Most frequent brand from past orders
const favBrand = mostFrequent(customer.orders.flatMap(o => o.items.map(i => i.product.brand_id)));

// Products they haven't ordered, same brand, currently in stock
const recs = products.filter(p => 
  p.brand_id === favBrand &&
  p.is_active &&
  p.stock_status === 'in_stock' &&
  !customer.orderedProductIds.has(p.id)
).slice(0, 4);
```

**Log impressions:** insert to `recommendation_events` with `event_type: 'shown'`. When clicked, insert with `event_type: 'clicked'`. Lets us tune recommendations later.

## /account/orders

### List view
- All orders, newest first, paginated
- Filters: status, date range, search by product name
- Each row: order number, date, total, status pill, item count, "Reorder" button

### /account/orders/[id]
- Full order breakdown
- Status timeline (paid → fulfilled → shipped → delivered)
- Items with images
- Shipping address
- Tracking numbers per vendor shipment (links to carrier tracking page)
- "Reorder this whole order" button → adds everything to cart at current prices (show notice if prices changed)
- "Reorder with edits" → takes them to cart pre-populated, can adjust
- "Print invoice" → PDF generation (v1 can be a simple print stylesheet; v2 could use PDFKit)
- "Contact us about this order" button → prefills a support form with order number

## /account/addresses

- List of saved addresses
- Add / edit / delete
- Mark default shipping, default billing
- Use during checkout for fast address selection

## /account/payment-methods

- List of saved cards (last 4 + brand + expiry only — never full PAN)
- Add new (launches Accept.js flow, tokenizes via CIM)
- Delete (removes from Auth.net CIM + Supabase)
- Cannot edit — delete + re-add

## /account/profile

- First name, last name, email (email change requires re-verification), phone
- Business info toggle: is_business, company_name, tax_id (optional)
- Change password
- Marketing opt-in toggle
- Delete account button (with confirmation modal) — v2: grandpa-approved process. V1: deactivate only.

## Login methods (both supported)

Supabase Auth handles both natively. Both always available on the login page.

### 1. Email + password (standard)
- Customer enters email + password
- "Forgot password" triggers reset-link email

### 2. Magic link (passwordless)
- Customer enters email only
- One-click login link sent to their inbox, valid for 15 minutes
- **Especially valuable for older customers** (grandpa's demographic) who forget passwords
- **The default path for migrated BigCommerce customers** on first login

### Login page UI
Two tabs or two buttons, clearly labeled:
```
┌─ Sign in ─────────────────────┐
│                                 │
│  ○ Email + password             │
│  ● Email me a login link        │
│                                 │
│  [ email ______________ ]       │
│  [       Send link       ]      │
│                                 │
└─────────────────────────────────┘
```

Default selection: magic link. Password stays a click away for customers who prefer it.

## Migrated customer first-login experience

Existing BigCommerce customers migrate over but **all need password reset** (we can't carry password hashes across platforms). With magic link available, this is painless.

On launch:
1. All migrated customer rows get `needs_password_reset = true`
2. Email blast to all migrated customers:
   > "Welcome to the new La Costa Gourmet site! Your account and order history are ready. Click the button below to log in — no password needed."
3. Button in email is a magic link → auto-logs them in
4. First login shows a welcome banner: "Your past orders are in the Orders section. You can also set a password anytime in your profile — or keep using magic links, whichever is easier."
5. In `/account/profile`, they can optionally set a password for future sessions

## Guest order tracking

For customers who checked out as guest, provide `/orders/track` — enter order number + email to see status and tracking. No login required.

## Mobile experience

- Product cards 1-column on mobile, 2-col on tablet, 3-4 on desktop
