# 02.1 вҖ” Architecture Overview

## Stack

| Layer | Choice | Why |
|---|---|---|
| Framework | **Next.js 14 App Router** | SSR for SEO, server components, generateMetadata per page. Asher knows this stack cold from Healthy Aminos. |
| Language | **TypeScript** (strict mode) | Same as HA. Catches schema drift at compile time. |
| Styling | **Tailwind CSS + CSS variables** | Variables for brand tokens, Tailwind for layout. No CSS-in-JS. |
| Database | **Supabase (Postgres 15)** | Auth + database + storage in one. Same as HA. |
| Auth | **Supabase Auth** | Email + password, with magic link option for existing customers' first login |
| File storage | **Supabase Storage** | Product images, vendor line sheet PDFs |
| Payments | **Authorize.net Accept.js** | Grandpa's existing gateway вҶ’ Wells Fargo merchant account. Hosted card fields = SAQ A-EP compliance. |
| Tax | **Settings-driven, $0 default** | Jeff's catalog is 100% non-taxable grocery food. Zero tax for v1. `tax.enabled` setting + per-product `tax_class` field give clean path to enable later if he adds taxable SKUs or hits nexus elsewhere. |
| Accounting | **QuickBooks Online API** | Direct export, not CSV |
| Transactional email | **Resend** | Order confirmations, shipping notifications to customers |
| Vendor email | **Google Workspace SMTP / Gmail API** | Sent from grandpa's actual company email address (not a no-reply) вҖ” so Houston's and other vendors see it come from him |
| Shipping rates v1 | **Flat-rate rules** | $12.99 under $70, free over $70, $25 HI/AK. Set in admin settings table. |
| Shipping rates v2 | **Shippo or EasyPost** | Real-time carrier rates. Deferred to phase 2. |
| Hosting | **Orange Website VPS (Iceland)** | Same as HA. Cheap, stable, Asher's existing infrastructure. |
| Process manager | **PM2** | Same as HA. Auto-restart, log management. |
| Reverse proxy | **Nginx** | SSL termination, static asset caching |
| DNS | **Cloudflare** | Recommended вҖ” free tier covers CDN + DDoS + SSL. Grandpa may already have DNS elsewhere; plan for migration. |
| Monitoring | **Sentry (free tier)** | Error tracking. Optional v1. |

## Why not X

- **Why not Shopify?** Considered. Rejected because grandpa wants full finance visibility and deep customizations, and Asher can build + deploy Next.js faster than configure Shopify + apps. Shopify also charges a 2% extra transaction fee if not using Shopify Payments (grandpa is locked into Auth.net + WF).
- **Why not stay on BigCommerce?** Grandpa wants a brand new site. The current theme is dated. BigCommerce Stencil customization is painful vs. Next.js.
- **Why not Vercel?** More expensive at scale. Orange VPS is already paid for. Also: bandwidth limits on Vercel's free tier are easy to hit on an e-commerce site with product images.
- **Why not Stripe?** Grandpa's merchant account with Wells Fargo is already underwritten, rates locked in, fraud rules configured. Migrating to Stripe means re-underwriting and losing his AFDS setup.
- **Why Supabase instead of Planetscale / Neon / etc?** Auth + DB + storage in one. Asher knows it from HA. RLS policies make admin/customer separation clean.

## Deployment

- Match Healthy Aminos VPS pattern: PM2 + Nginx + Let's Encrypt
- Staging: `staging.lacostagourmet.com` subdomain on same VPS, separate PM2 process on different port
- Production cutover: change DNS A record from BigCommerce to VPS, update Nginx config, confirm SSL cert provisioned
- Database migrations: Supabase CLI, run before deploy

See `07-launch/` for full deployment + cutover plan.
