# 01.3 — Design System (v5 Cinematic)

**This is the authoritative visual language for La Costa Gourmet.** Every page on the site follows the rules in this document. Claude Code reads this before building any UI.

The approved reference is the v5 cinematic homepage mockup from the planning conversation — warm mercantile, real photography, Fraunces + JetBrains Mono, italic accents, staggered animations, catalog-style numbering.

---

## Design Principles

1. **Warm mercantile, not modern DTC.** This site sells bulk cafe supplies to independent coffee shops. It should feel like a trusted printed catalog, not a lifestyle brand.
2. **Typography does the heavy lifting.** Bold serif display, precise mono labels. No sans-serif body text.
3. **Italic for emphasis.** Key nouns in headlines are italicized in brand-deep brown. Creates rhythm and hierarchy.
4. **Catalog conceits everywhere.** Section numbers (§ I, § II), roman numerals, dispatch references (№ 0042, Vol. XXII), SKUs visible on products, founding years on brands.
5. **Animation is intentional, never decorative.** Page-load stagger. Hover states that reward the pointer. No scroll-jacking. No bouncing. No spinning. No glow pulses except the single "live" dot on the hero image.
6. **Photography is color-graded uniformly.** Warm tones, slightly desaturated, subtle contrast bump. Never use raw stock photos without the grade applied.

### Typographic scale

Apply these sizes exactly. No custom deviations.

| Element | Class | Size | Line-height | Weight | Style | Color |
|---|---|---|---|---|---|---|
| Hero headline (h1) | `.type-display-1` | 92px | 0.93 | 400 | serif | ink |
| Section headline (h2) | `.type-display-2` | 48px | 1.02 | 400 | serif | ink |
| Story h3 | `.type-display-3` | 62px | 1.02 | 400 | serif | cream |
| Brand row name | `.type-brand` | 22px | 1.1 | 500 | serif italic | brand-deep |
| Product price | `.type-price` | 22px | 1 | 500 | serif italic | brand-deep |
| Product name | `.type-product` | 15px | 1.4 | 400 | serif | ink |
| Body | `.type-body` | 16-17px | 1.65 | 400 | serif | ink-2 |
| Mono label / eyebrow | `.type-label` | 10-11px | — | 400 | mono uppercase | varies |
| Section num (Roman) | `.type-numeral` | 36px | 1 | 400 | serif italic | brand-deep |

**Letter-spacing rules:**
- Display headlines (48px+): `-0.028em` to `-0.035em`
- Body text: normal
- Mono labels: `0.18em` to `0.26em` (scale with prominence)
- Italic accent words in headlines: slightly tighter than the surrounding text (`-0.02em`)

**Italic usage — follow this pattern:**

In every headline, one key noun should be italicized and colored in `--brand-deep`. This creates the mercantile editorial feel. Examples:

- "Shop by *category*."
- "New *arrivals*."
- "Twelve brands, *one roof*."
- "Volume pricing, *reliable supply*."
- "Twenty-two years of the *same* promise."
- "The pantry for cafés, *kitchens* & home."

**No italics in body copy.** Italic is reserved for headlines, drop caps, product prices, brand names in the directory, and the gold drop-cap W in the story.

**Roman numerals in Fraunces italic** appear in every section marker:

```html
<div class="section-num">
  <span class="rom">I</span>
  The Departments
</div>
```


### Color use rules

- **Body background:** `--color-paper`
- **Nav background:** `--color-cream`
- **Section alternation:** paper → paper-2 → paper → dark story → paper
- **Never pure white.** The lightest surface is `--color-cream`. Pure white reads wrong against the warm system.
- **Never pure black.** The darkest is `--color-ink`.
- **Italic accent words:** `--color-brand-deep`
- **Eyebrows, "Just In" stickers, hero line rules:** `--color-accent`
- **Gold usage:** only on dark backgrounds (story section, footer, B2B) and as hover states. Never gold on paper — it looks muddy.

## Spacing & Layout

### Page grid

- Outer padding: **32px** horizontal on all sections (desktop). On mobile drop to 20px.
- Max content width: **1280px**, centered. Add `max-width: 1280px; margin: 0 auto;` at section level.
- Section vertical rhythm: **72px top / 56px bottom** for standard sections. **80-96px** for hero and story. Never less than **56px** between sections.

### Component spacing

- Card internal padding: **18px**
- Button padding: **16px 26px** (solid), **14px 22px** (outline smaller use)
- Section head bottom margin: **44px**
- Between section head and content: **20px** gap via border-bottom

### Borders & rules

- Default rule: `1px solid var(--rule)`
- Emphasis rule: `1px solid var(--rule-strong)`
- Dark mode rules: `1px solid rgba(246, 238, 222, 0.14)`
- Dashed rules used only on: product card `--foot` separator, brand-row subtle accents, list item dividers inside dark panels

No rounded corners on most things — the catalog feel wants sharp 90-degree rectangles. Exceptions:
- Outer page container / card grouping: `border-radius: 6px`
- Cart total badge (`.cart-dot`): no radius, intentionally rectangular
- Image overlays (blurred-glass): no radius

### Scroll-triggered reveals (use IntersectionObserver)

For sections below the fold, wrap with `data-reveal` and use one small client component:

```tsx
// components/Reveal.tsx
'use client';
import { useEffect, useRef, useState } from 'react';

export function Reveal({ children, delay = 0, className = '' }: {
  children: React.ReactNode;
  delay?: number;
  className?: string;
}) {
  const ref = useRef<HTMLDivElement>(null);
  const [visible, setVisible] = useState(false);

  useEffect(() => {
    if (!ref.current) return;
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setVisible(true);
          observer.disconnect();
        }
      },
      { threshold: 0.15, rootMargin: '0px 0px -60px 0px' }
    );
    observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  return (
    <div
      ref={ref}
      className={className}
      style={{
        opacity: visible ? 1 : 0,
        transform: visible ? 'translateY(0)' : 'translateY(24px)',
        transition: `opacity 0.9s cubic-bezier(0.2, 0.8, 0.2, 1) ${delay}s, transform 0.9s cubic-bezier(0.2, 0.8, 0.2, 1) ${delay}s`,
      }}
    >
      {children}
    </div>
  );
}
```

Use `<Reveal delay={0.1}>`, `<Reveal delay={0.2}>` etc. to stagger reveals within a section.

### Hover transitions — follow these patterns exactly

**Button solid (`.btn-solid`):**
- Background overlay slides up from bottom on hover, switching from ink to brand-deep
- 0.3s duration, `--ease-out`

**Button outline (`.btn-outline`):**
- Full fill transition (background + text color invert)
- Arrow character `→` translates 4px right
- 0.3s duration

**Nav links:**
- Bottom border `scaleX(0)` → `scaleX(1)`, `transform-origin: right center` on enter, `left center` on stick
- 0.35s duration

**Product cards:**
- `translateY(-6px)`
- Dual box-shadow: inked offset `4px 8px 0 rgba(26, 17, 10, 0.08)` + soft `8px 16px 24px rgba(26, 17, 10, 0.12)`
- Border changes to solid ink
- Image inside zooms `scale(1.06)` over 0.7s
- Image gradient overlay intensifies slightly
- All on 0.35s `--ease-out`

**Brand rows:**
- Left-side accent bar `scaleY(0)` → `scaleY(1)` in accent color, top origin
- Row slides right `padding-left: 28px` (from 22)
- Background transitions to paper-2
- 0.3s duration

**Category tiles:**
- Image scales `1.08` and saturation lifts from 0.7 to 1
- Body background stays still
- 0.8s image transition (longer, more cinematic)

**Newsletter input:**
- Border color transitions to `--color-gold-bright` on focus
- Background alpha lifts slightly
- 0.2s duration

### Ticker strip

Infinite horizontal scroll. 40s linear loop. Content is duplicated once inside `.ticker-track` so the seam is invisible. `overflow: hidden` on wrapper.

### Never use

- `animation-name: bounce`
- `animation-name: shake`
- Rotation of full UI elements (only the tiny `◆` or `✺` characters may rotate subtly)
- CSS `transform: rotate3d()`
- `scroll-behavior: smooth` globally (Safari has issues)
- JS scroll-jacking libraries (no Locomotive, no Lenis smooth scroll)
- Hover effects on mobile (`@media (hover: hover)` wrap all hover rules)

## Image Handling

### Grading (critical — apply site-wide)

Every photograph on the site gets this CSS filter:

```css
.img-warm {
  filter: saturate(0.80) contrast(1.08) brightness(0.95);
}

.img-story {
  /* deeper, more editorial */
  filter: saturate(0.6) contrast(1.15) brightness(0.85) sepia(0.15);
}

.img-product {
  filter: saturate(0.8) contrast(1.08) brightness(0.95);
}

.img-hero {
  filter: saturate(0.85) contrast(1.05) brightness(0.92);
}
```

All product photos, category imagery, and hero photos get this treatment. It's what makes disparate photography feel like one brand.

### Overlays

Every hero and product image has a subtle radial gradient overlay to give depth:

```css
.img-overlay-radial::after {
  content: "";
  position: absolute;
  inset: 0;
  background: radial-gradient(circle at 20% 30%, transparent 40%, rgba(26, 17, 10, 0.4) 100%);
  pointer-events: none;
}
```

### Photography sources

**V1 launch:** use Jeff's existing BC product images (we have URLs from the export). Download them to Supabase Storage, apply the warm grading via CSS at render time.

**Hero / category / story imagery:** use high-quality Unsplash or Pexels images, grade them, save to Supabase Storage. Specific queries:
- Hero: `chai latte steam ceramic cup` warm lighting
- Category tiles: individual shots per category — tea leaves, cocoa powder, frappe glass, oatmeal bowl, smoothie, syrup bottles
- Story: cafe counter / coffeehouse interior, warm natural light

**Phase 2:** commission a basic product shoot at Jeff's kitchen (2 hours, good window light, 30 SKUs on a kraft paper backdrop). Replace stock.

### Image sizing

- Hero: 4:5 aspect ratio (e.g., 800 × 1000)
- Category tiles: 1:1 cropped to 600px, rendered at `height: 140px`
- Product cards: square crop, rendered at `height: 220px`
- Story: 3:4 aspect ratio

All images should be served via `next/image` with explicit `width` and `height` attributes to prevent layout shift.

Structure:
1. TopRail + Nav

### `<AccountPage>` — customer dashboard

Structure:
1. TopRail + Nav
2. **PageHeader** — "Welcome back, [First]." with subscriber-since date in mono
3. 2-column: sidebar navigation left (240px) / content right. Sidebar uses `<BrandRow>`-style hover (left bar + slide).

Admin is functional, not decorative. Simpler palette (heavier use of paper-2 for density), but keeps the type system. See `04.3-admin-dashboard.md` for specifics. The design system still applies — mono labels, italic accents, catalog numbering on sections.

## Mobile Adaptations

### Breakpoints

```
Mobile:   <640px
Tablet:   640px - 1023px
Desktop:  1024px+
```

### Mobile rules

- Nav collapses to: logo left, hamburger right. Promo rail shortens to shipping-only message.
- Hero: stack image on top, text below. `h1` drops to 54px.
- Section heads: numeral + title stack vertically. Drop the "View All" link into the body.
- Category grid: 2 columns on mobile.
- Product grid: 2 columns on mobile, 1 below 420px.
- Brand directory: 2 columns.
- Ticker: stays, but 14px font to keep readable.
- Story section: stack vertically, image first. h3 drops to 40px.
- B2B band: stack vertically, summary card first.
- Footer: 1 column, accordion-style if desired.
- Turn off all hover effects (`@media (hover: hover)` gates).

## Accessibility

- Every image needs real `alt` text (not "image" or filename)
- Color contrast: test every mono label — `ink-muted` on `paper` can fall below 4.5:1. Verify with axe.
- Keyboard focus states: every interactive element gets a visible focus ring. Use `outline: 2px solid var(--color-gold-bright); outline-offset: 3px;` — it fits the brand.
- Reduced motion: wrap all animations in:
  ```css
  @media (prefers-reduced-motion: no-preference) { /* animations here */ }
  ```
- Screen reader: every section gets an `<h2>` — don't skip heading levels for visual reasons.

## The Final Word

Jeff's customers are B2B buyers who reorder the same SKUs every 6 weeks. They don't need pretty — they need **fast, legible, trustworthy**. This design system looks expensive because **every decision was intentional**, not because it's loud.

If Claude Code starts building a page and wants to deviate from any of this — add a gradient, use a different font, round every corner, add a glow — the answer is: read this doc again, pick the closest sanctioned pattern, and use that.

The consistency is the value.
