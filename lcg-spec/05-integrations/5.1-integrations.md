## Authorize.net (Accept.js + API)

**Package:** `authorizenet` npm package + Accept.js script tag on checkout page.

**Files to create:**
- `lib/authnet/client.ts` — typed wrapper around authorizenet SDK
- `lib/authnet/accept-nonce.ts` — create transaction from Accept.js nonce
- `lib/authnet/cim.ts` — customer profile + payment profile tokenization for saved cards
- `lib/authnet/webhook.ts` — signature verification + event handler
- `lib/authnet/afds-codes.ts` — map response codes to our internal status

**Key gotchas:**
- Auth.net responseCode `1` means "approved" BUT if it has a fraud message, it's actually held. Check both.
- Webhook signature verification uses HMAC-SHA512 with the signature key, comparing `X-ANET-Signature` header.
- Sandbox: use `https://apitest.authorize.net/xml/v1/request.api`. Production: `https://api.authorize.net/xml/v1/request.api`.
- Accept.js `data-client-key` is the public client key; different from API login / transaction key. It's safe to expose.

**Env:**
```
AUTHNET_API_LOGIN_ID
AUTHNET_TRANSACTION_KEY
AUTHNET_SIGNATURE_KEY       # webhook HMAC
NEXT_PUBLIC_AUTHNET_CLIENT_KEY
AUTHNET_ENVIRONMENT=production|sandbox
```

## Tax calculation — zero by default (TaxJar is v2-optional)

**V1 reality:** Jeff's entire catalog is non-taxable grocery food in California (his only nexus state). All orders return tax = $0. Do NOT integrate TaxJar in v1.

**What to build:**
- Settings: `tax.enabled` (boolean, default `false`)
- Products: nullable `tax_class` column (TEXT) — when null, product is non-taxable
- `lib/tax/calculate.ts` — returns `0` unless (a) `tax.enabled` is true, AND (b) cart contains products with a `tax_class`, AND (c) ship-to state is in `settings.tax.nexus_states`
- Admin product edit page: "Is this item taxable?" toggle, default off. If toggled on, shows a "Tax class" dropdown (states can be added in settings).

**If Jeff ever needs real tax calc (phase 2 trigger):**
- He adds a taxable SKU (candy, merchandise, prepared food), OR
- Sales in a new state cross ~$100K/year (economic nexus threshold)

Then wire up TaxJar:
- Add `lib/tax/taxjar-client.ts` implementing the same `calculate(cart, address)` interface
- Flip `tax.enabled = true`
- Tag taxable products with a TaxJar tax code
- Test, deploy

**Why this approach:** saves $19/month on something that would always return $0, but leaves a clean path to add real tax calculation later in ~2 hours of work.

## Google Workspace (vendor emails from grandpa's account)

The subtle one. Customer emails go through Resend (from orders@lacostagourmet.com, a service address). **Vendor emails go through grandpa's real Google Workspace account** so Houston's and others see them as coming from him personally.

**Two implementation options — pick based on grandpa's comfort:**

### Option A: App Password (simpler)
- Grandpa enables 2FA on his Google account (he may already have this — Workspace admins often enforce it)
- He generates an "App Password" at `myaccount.google.com/apppasswords`
- We use Gmail SMTP (`smtp.gmail.com:587`) with his email + app password
- Drawbacks: he has to share the app password with us; less secure than OAuth
- Good for: fast setup, if grandpa is non-technical

### Option B: Gmail API OAuth (better)
- We register a Google Cloud project with Gmail API scope
- Grandpa OAuth-authorizes our app once via consent screen
- We store a refresh token; send via Gmail API `users.messages.send`
- Sent emails appear in his Gmail Sent folder naturally
- Drawbacks: more setup, Google verifies apps that use sensitive scopes (workable for single-user)

**Recommended: Option B for long-term, Option A to start and migrate.**

**Files:**
- `lib/workspace/gmail-client.ts`
- `lib/workspace/send-vendor-email.ts`

**Rate limits:**
- Gmail SMTP: 500 recipients/day for free, 2000/day for Workspace
- Gmail API: 250 quota units per user per second (sending = 100 units) — way more than needed

**Env (Option A):**
```
GMAIL_SMTP_USER=grandpa@lacostagourmet.com
GMAIL_SMTP_APP_PASSWORD=
```

**Env (Option B):**
```
GMAIL_OAUTH_CLIENT_ID
GMAIL_OAUTH_CLIENT_SECRET
GMAIL_OAUTH_REFRESH_TOKEN
```
